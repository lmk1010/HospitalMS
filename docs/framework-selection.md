# 医院放疗管理系统复刻：后端框架选型结论

## 现有系统截图可见模块
- 患者管理：患者登记、患者列表
- 定位管理：CT预约、CT排队叫号
- 勾画管理：靶区勾画、勾画审核
- 计划管理：计划申请、计划设计、组长审核、医师审核、计划复核、计划验证
- 治疗管理：治疗预约、排队叫号、治疗执行、治疗小结
- 费用管理：费用相关模块

## 最终建议
### 推荐方案：`Yudao Boot Mini` / `ruoyi-vue-pro` 单体版
原因：
- Spring Boot 3 + Vue3 + MySQL 路线成熟
- 单体项目启动快，适合你现在快速复刻
- 自带 RBAC、代码生成、工作流、文件上传、定时任务等后台常用能力
- 比纯低代码平台更适合后续做医院业务深度定制

## 不同框架结论
### 1. `Yudao Boot Mini`
- 适合：想尽快上线，同时后续继续长期迭代
- 优点：文档全、社区活跃、生成器强、工程结构规整
- 结论：最适合你这次项目

### 2. `JeecgBoot`
- 适合：要最快做出 CRUD/表单/审批演示版
- 优点：低代码能力强，报表和流程能力现成
- 缺点：后期深度定制成本通常更高
- 结论：如果只追求最短时间出 demo，它可能更快；但不建议作为你这次的首选

### 3. `RuoYi-Vue-Plus`
- 优点：现代化、单体可做、也有代码生成
- 缺点：综合资料和落地案例没有 Yudao 顺手
- 结论：可作为备选，不是第一选择

### 4. `RuoYi-Vue`
- 优点：经典、资料多
- 缺点：相对老，后续你还得自己补不少现代能力
- 结论：不如直接选 Yudao

### 5. `SpringBlade`
- 优点：企业化能力强
- 缺点：整体更重，不是“最快复刻单体项目”的最佳路线
- 结论：不推荐作为本项目首选

## 当前建议目录
- `backend/`：Java Spring 单体后端
- `frontend/`：Vue3 前端
- `sql/`：MySQL 初始化脚本与迁移
- `docs/`：原系统分析、接口清单、原型说明

## 一句话决策
这次项目要“快、稳、后面还能改”，后端直接选 `Yudao Boot Mini` 最合适。
