# 上海XX医院管理平台实施计划

## 1. 目标
- 以 `Yudao Boot Mini` 单体架构为底座，快速复刻并重做医院放疗管理系统
- 保留 `RBAC + BPM + 文件 + 消息 + 日志 + 代码生成` 基础能力
- 一期先打通 `患者 + 科室 + 医生 + 流程入口 + 菜单骨架`
- 后续按业务链路逐步补齐 `定位 -> 勾画 -> 计划 -> 治疗 -> 费用`

## 6. 后端实施策略
### 6.1 模块结构
- 新建业务模块：`backend/yudao-module-hospital`
- 包路径建议：`cn.iocoder.yudao.module.hospital`
- 子域按业务分包：`patient`、`doctor`、`dept`、`ct`、`contour`、`plan`、`treatment`、`fee`

### 6.2 开发顺序
1. 先建表
2. 用代码生成器生成 `Dept / Doctor / Patient` CRUD
3. 再接菜单、权限、前端页面
4. 再接 BPM 表单和流程定义
5. 最后补统计、排队、看板类页面

## 7. 前端实施策略
- 前端目录统一放在：`frontend/apps/web-antd/src/views/hospital/`
- 一期先做骨架页，确认菜单层级和字段布局
- 二期替换成真实接口、表格、表单、详情页
- 首页工作台改成医院业务待办和流程入口

## 10. 结论
这套系统确实是大体系，不适合一次性全写完。正确做法是：
- 先把菜单、表结构、流程边界定下来
- 再按 `患者 -> 医生 -> 定位 -> 计划 -> 治疗 -> 费用` 顺序推进
- 这样你能很快看到成果，也不会后面推翻重做
